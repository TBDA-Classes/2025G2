MAIN QUERIES

This section documents a collection of key SQL queries used during the analysis, exploration, and validation phases of the project.

These queries were used to: - Understand raw machine behavior - Validate ETL logic - Derive utilization, energy, and alert metrics - Prototype aggregations later implemented in ETL scripts

—

## Machine Utilization Queries

### Machine Utilization (Percentages)

This query calculates daily machine utilization percentages by classifying 10-minute activity blocks into operational states and aggregating them per day.

 1WITH RangoTiempo AS (
 2    SELECT
 3        '2021-01-07 00:00:00'::timestamp AS inicio,
 4        '2021-01-16 00:00:00'::timestamp AS fin
 5),
 6ResumenFloat AS (
 7    SELECT
 8        to_timestamp(
 9            floor(
10                extract(epoch from to_timestamp(TRUNC(CAST(date AS bigint)/1000))) / 600
11            ) * 600
12        ) AS inicio_bloque,
13        COUNT(*) AS cnt,
14        MIN(date) AS min_ms,
15        MAX(date) AS max_ms
16    FROM variable_log_float
17    WHERE to_timestamp(TRUNC(CAST(date AS bigint)/1000))
18          BETWEEN (SELECT inicio FROM RangoTiempo)
19          AND (SELECT fin FROM RangoTiempo)
20    GROUP BY 1
21),
22ResumenString AS (
23    SELECT
24        to_timestamp(
25            floor(
26                extract(epoch from to_timestamp(TRUNC(CAST(date AS bigint)/1000))) / 600
27            ) * 600
28        ) AS inicio_bloque,
29        COUNT(*) AS cnt,
30        MIN(date) AS min_ms,
31        MAX(date) AS max_ms
32    FROM variable_log_string
33    WHERE to_timestamp(TRUNC(CAST(date AS bigint)/1000))
34          BETWEEN (SELECT inicio FROM RangoTiempo)
35          AND (SELECT fin FROM RangoTiempo)
36    GROUP BY 1
37),
38TotalActivity AS (
39    SELECT
40        COALESCE(f.inicio_bloque, s.inicio_bloque) AS inicio_bloque,
41        (COALESCE(f.cnt, 0) + COALESCE(s.cnt, 0)) AS cambios_totales,
42        LEAST(f.min_ms, s.min_ms) AS min_ts_real,
43        GREATEST(f.max_ms, s.max_ms) AS max_ts_real
44    FROM ResumenFloat f
45    FULL OUTER JOIN ResumenString s
46        ON f.inicio_bloque = s.inicio_bloque
47),
48GrillaBloques AS (
49    SELECT generate_series(
50        (SELECT inicio FROM RangoTiempo),
51        (SELECT fin FROM RangoTiempo) - interval '10 minutes',
52        '10 minutes'
53    ) AS inicio_bloque
54),
55Clasificacion AS (
56    SELECT
57        g.inicio_bloque,
58        COALESCE(t.cambios_totales, 0) AS cambios,
59        CASE
60            WHEN COALESCE(t.cambios_totales, 0) = 0 THEN 'PARADA'
61            WHEN COALESCE(t.cambios_totales, 0) <= 30 THEN 'IDLE'
62            ELSE 'OPERACION'
63        END AS estado,
64        CASE
65            WHEN COALESCE(t.cambios_totales, 0) = 0
66                 THEN g.inicio_bloque
67            ELSE to_timestamp(t.min_ts_real / 1000.0)
68        END AS start_ts,
69        CASE
70            WHEN COALESCE(t.cambios_totales, 0) = 0
71                 THEN g.inicio_bloque + interval '10 minutes'
72            ELSE to_timestamp(t.max_ts_real / 1000.0)
73        END AS end_ts
74    FROM GrillaBloques g
75    LEFT JOIN TotalActivity t
76        ON g.inicio_bloque = t.inicio_bloque
77)
78SELECT
79    date(dia_referencia) AS fecha,
80    estado,
81    ROUND(
82        SUM(EXTRACT(EPOCH FROM (fin_corte - inicio_corte))) / 3600.0,
83        2
84    ) AS tiempo_horas,
85    ROUND(
86        (SUM(EXTRACT(EPOCH FROM (fin_corte - inicio_corte))) / 86400.0) * 100,
87        2
88    ) AS porcentaje_dia
89FROM Clasificacion;

## Program History Queries

### Program History (24 Hours)

Computes daily program execution durations by reconstructing program state intervals and cutting them by calendar day.

 1WITH RangoTiempo AS (
 2    SELECT
 3        '2021-01-07 00:00:00'::timestamp AS inicio,
 4        '2021-01-16 00:00:00'::timestamp AS fin
 5),
 6EstadoPrevio AS (
 7    SELECT
 8        (SELECT inicio FROM RangoTiempo) AS dt,
 9        CAST(value AS integer) AS estado_numerico
10    FROM variable_log_float
11    WHERE id_var = 581
12      AND to_timestamp(TRUNC(CAST(date AS bigint)/1000))
13          < (SELECT inicio FROM RangoTiempo)
14    ORDER BY date DESC
15    LIMIT 1
16),
17LogsRango AS (
18    SELECT
19        to_timestamp(TRUNC(CAST(date AS bigint)/1000)) AS dt,
20        CAST(value AS integer) AS estado_numerico
21    FROM variable_log_float
22    WHERE id_var = 581
23      AND to_timestamp(TRUNC(CAST(date AS bigint)/1000))
24          BETWEEN (SELECT inicio FROM RangoTiempo)
25          AND (SELECT fin FROM RangoTiempo)
26)
27SELECT
28    date(dia_referencia) AS fecha,
29    estado_numerico,
30    ROUND(
31        SUM(EXTRACT(EPOCH FROM (fin_real - inicio_real))),
32        2
33    ) AS duracion_segundos
34FROM LogsRango
35GROUP BY fecha, estado_numerico;

## Temperature Queries

### Temperature History (24 Hours)

Retrieves raw temperature sensor readings for a set of temperature-related variables over a given time window.

 1SELECT
 2  b.name AS variable,
 3  to_timestamp(CAST(a.date AS bigint) / 1000) AS dt,
 4  a.value
 5FROM variable_log_float a
 6JOIN variable b ON a.id_var = b.id
 7WHERE to_timestamp(CAST(a.date AS bigint) / 1000)
 8      BETWEEN '2021-01-04 13:39:30+01'
 9      AND '2021-01-05 22:00:00+01'
10  AND b.name IN (
11    'TEMPERATURE_BASE',
12    'TEMPERATURA_BASE',
13    'TEMPERATURE_HEAD'
14  )
15ORDER BY variable, dt;

## Energy Consumption Queries

### Hourly Consumption

Computes hourly energy consumption (kWh) based on motor utilization percentages and nominal power.

1SELECT
2  date_trunc('hour', ts) AS hour_ts,
3  ROUND(SUM(energy_kwh)::numeric, 3) AS energy_kwh
4FROM segments_energy
5GROUP BY date_trunc('hour', ts)
6ORDER BY hour_ts;

—

### Energy by Shift

Aggregates energy consumption by production shift (Day, Swing, Night).

1SELECT
2  shift_name,
3  ROUND(SUM(energy_kwh)::numeric, 3) AS energy_kwh
4FROM by_shift
5GROUP BY shift_name
6ORDER BY shift_name;

## Alarm Queries

### Alarm Count by Category

Counts alarms grouped by severity category.

1SELECT
2    category,
3    COUNT(*) AS total_alerts
4FROM categorized
5GROUP BY category;

—

### Alarm Count by Shift and Category

Counts alarms per shift and severity category.

1SELECT
2    shift,
3    COUNT(*) AS total_alerts
4FROM categorized
5GROUP BY shift
6ORDER BY shift;

—

### Full List of Alerts

Retrieves a chronological list of alarms with classification.

1SELECT
2    category AS alert_type,
3    alarm_code,
4    timestamp
5FROM categorized
6ORDER BY timestamp DESC;

—

### Alarm Detail for a Specific Code

Filters alarms for a specific alarm code and date.

1SELECT
2    alarm_code,
3    timestamp,
4    alarm_description
5FROM alarm_data
6WHERE alarm_code = :alarm_code
7  AND timestamp::date = :target_date;
